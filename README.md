# IOTA5505 EM-GMM Reproduction

This repository contains the MATLAB code and fixed-seed numerical outputs for the Gaussian mixture model reproduction experiment in the mini-project report:

**From Incomplete Data to Maximum Likelihood: A Mini-Project Report on the EM Algorithm**

The experiment illustrates the expectation--maximization (EM) algorithm for maximum-likelihood estimation with incomplete data. The numerical reproduction specializes the general EM framework to a synthetic two-dimensional Gaussian mixture model (GMM) with known ground-truth parameters. The EM algorithm is run using only the observed data, while the true latent component labels are retained only for post-hoc diagnostic evaluation.

The repository is intended to support reproducibility of the numerical part of the report. It includes:

1. a self-contained MATLAB implementation of the GMM--EM algorithm;
2. a fixed-seed archive of the numerical outputs used in the report.

## Files

### `Reproduce_GMM_EM_Mini_Project.m`

This is the main MATLAB script. It performs the full numerical reproduction experiment, including:

- generation of synthetic two-dimensional GMM data from fixed ground-truth parameters;
- EM initialization from fixed random seed;
- computation of posterior responsibilities in the E-step;
- weighted Gaussian maximum-likelihood updates in the M-step;
- stable observed log-likelihood evaluation using a log-sum-exp computation;
- covariance symmetrization and positive-definiteness regularization;
- full recording of the observed log-likelihood trace;
- full recording of the EM parameter trajectory;
- independent recomputation of the closed-form theoretical M-step at every iteration;
- update-consistency comparison between the stored numerical update and the independently recomputed formula update;
- final label matching between estimated and true components;
- final comparison of weights, means, and covariance matrices with the ground truth;
- post-hoc hard-label accuracy evaluation;
- generation of CSV tables, LaTeX table fragments, and figures.

The script is self-contained and does not require external input data. It generates the synthetic data internally using the fixed random seed specified in the script.

The implementation does not rely on the MATLAB Statistics and Machine Learning Toolbox. It uses custom local functions for Gaussian log-density evaluation, categorical sampling, log-sum-exp stabilization, EM updates, component matching, and table generation.

## Numerical Experiment

The experiment uses a controlled synthetic Gaussian mixture model with known population parameters. The observed data consist only of the generated two-dimensional samples. The latent labels are not used by the EM algorithm.

### Basic Configuration

```text
Sample size:              n = 900
Dimension:                d = 2
Number of components:     K = 3
Data-generation seed:     103
Initialization seed:      1
Number of random starts:  1
Maximum EM iterations:    200
Stopping tolerance:       1e-8
Covariance ridge:         1e-6
```

### Ground-Truth Mixing Weights

```text
pi* = (0.35, 0.40, 0.25)
```

### Ground-Truth Means

```text
mu_1* = (-4, 0)
mu_2* = ( 0, 4)
mu_3* = ( 4, 0)
```

### Ground-Truth Covariance Matrices

```text
Sigma_1* = [0.70   0.20;
            0.20   0.50]

Sigma_2* = [0.60  -0.15;
           -0.15   0.80]

Sigma_3* = [0.90   0.25;
            0.25   0.70]
```

The true labels are generated together with the data but are used only after the EM procedure has finished. They are used for the final hard-label diagnostic and do not enter the EM updates.

## How to Run

Clone or download the repository, open MATLAB in the repository root directory, and run:

```matlab
run("Reproduce_GMM_EM_Mini_Project.m")
```

Alternatively, from a system terminal with MATLAB available on the path, run:

```bash
matlab -batch "run('Reproduce_GMM_EM_Mini_Project.m')"
```

The script creates an output directory named

```text
gmm_em_results/
```

and writes the regenerated numerical outputs into that directory.

```text

Initial observed log-likelihood:             -4744.02351571491
Final observed log-likelihood:               -3118.19614126235
Smallest likelihood increment:               7.34871719032526e-10
```

The observed log-likelihood is monotone non-decreasing in the reported run. This numerically supports the standard EM monotonicity principle: an exact or generalized EM update that increases the expected complete-data log-likelihood cannot decrease the observed-data log-likelihood.

## Covariance Ridge and Update-Consistency Convention

The covariance update includes a small positive-definiteness regularization step. Specifically, the script symmetrizes each covariance matrix and applies the ridge parameter

```text
ridge = 1e-6.
```

The update-consistency check uses the same covariance post-processing convention on both sides of the comparison.

At each EM transition, the script computes the next parameters in two ways:

1. the implemented EM M-step computes the numerical update and applies covariance symmetrization and ridge regularization;
2. the independent closed-form reference update substitutes the same responsibility matrix into the theoretical weighted GMM M-step formulas and then applies the same covariance symmetrization and ridge regularization.

Therefore, the reported update discrepancies compare the stored implemented update with the corresponding regularized closed-form reference update. This explains why the covariance update discrepancy is at floating-point roundoff level rather than at the ridge scale.

In the fixed-seed run, the maximum overall discrepancy is approximately

```text
1.6012e-14.
```

This confirms that the MATLAB implementation is consistent with the theoretical EM recursions used in the report.

## Label Matching Convention

Finite mixture models are invariant under permutations of component labels. Therefore, component labels produced by EM have no intrinsic meaning.

For final interpretation, estimated components are matched to the ground-truth components by minimizing the total squared distance between estimated and true means. The reported final weights, means, covariance matrices, and hard-label accuracy are all computed after this label matching step.

The post-hoc hard-label accuracy is computed by assigning each observation to the component with the largest final responsibility. The reported value of `1.0000` reflects the well-separated structure of this synthetic example. It is a diagnostic result for this controlled experiment, not a general theoretical guarantee for EM.

## Reproducibility Notes

The experiment is deterministic once the following settings are fixed:

```text
dataSeed = 103
initSeed = 1
numStarts = 1
```

Minor differences in the last few decimal places may occur across MATLAB versions or operating systems because of floating-point arithmetic and linear algebra implementation details. Such differences should remain at numerical roundoff level and should not affect the main conclusions:

- the observed log-likelihood is monotone non-decreasing;
- the final fitted mixture recovers the intended three-component structure up to finite-sample variation and label switching;
- the update-consistency errors remain near machine precision.
